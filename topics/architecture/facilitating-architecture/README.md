# Facilitating Software Architecture Learning Path

This directory contains resources for learning the practices around facilitating software architecture using the book 'Facilitating Software Architecture' by Andrew Harmel-Law

## üéØ What is it? 

Facilitating Software Architecture is about leading collaborative architectural decision-making processes.  It focuses on creating environments where teams can make effective architectural choices together, emphasizing facilitation skills, workshop techniques, and collaborative practices over traditional top-down architecture approaches.

Key aspects include:
- **Collaborative Decision-Making**: Bringing diverse perspectives together for better architectural outcomes
- **Facilitation Skills**:  Leading architecture discussions without dominating them
- **Workshop Techniques**: Structured activities to explore, evaluate, and decide on architectural approaches
- **Team Empowerment**: Enabling teams to own their architectural decisions
- **Visualizing Architecture**: Making architectural concepts accessible and discussable

### Key Practices

- [ ] **Architecture Decision Records (ADRs)**: Document architectural decisions with context and consequences
- [ ] **Architecture Katas**: Team exercises to practice architectural thinking
- [ ] **Event Storming**: Collaborative domain modeling technique
- [ ] **C4 Modeling**: Context, Containers, Components, and Code diagrams
- [ ] **Quality Attribute Workshops**: Identifying and prioritizing non-functional requirements
- [ ] **Trade-off Analysis**: Systematically evaluating architectural options
- [ ] **Mob/Ensemble Architecture**: Collaborative real-time architecture design
- [ ] **Architecture Review Sessions**: Structured peer reviews of architectural decisions
- [ ] **Facilitation Patterns**: Repeatable techniques for guiding discussions
- [ ] **Visual Collaboration**: Using diagrams, sketches, and models to align understanding

- [ ] Create new facilitation techniques and patterns
